# 작업중지권 진성/가성 판정 정책 v1

## 최종 출력

반드시 아래 JSON 형식으로만 출력한다.

```json
{
  "판정": "진성 또는 가성",
  "판단근거": "적용된 Step번호와 핵심 사유",
  "확신도": 0,
  "review_needed": false
}
```

## 정의

진성:
작업 진행 중 예측하지 못한 긴급한 물리적 위험, 즉 중대재해 임박 위험이 발생했고, 표준 안전수칙만으로 통제하기 어려워 작업자가 주체적으로 공정을 중단한 경우.

가성:
작업 전 점검사항, 일상적 안전수칙 위반, 행정/절차 이슈, 계획된 정비, 협력회사 자체 시정조치 등 작업중지권 행사의 긴급성과 필요성이 부족한 경우.

## Step 1. 진성 필수조건

아래 조건이 강하게 확인될 때 진성으로 판정한다.

1. 작업이 실제 진행 중이었다.
2. 예측하지 못한 긴급 상황이었다.
3. 사람에게 즉시 위해를 줄 수 있는 물리적 위험이었다.
4. 중대재해 임박성이 있었다.
5. 일반적인 표준 안전수칙이나 사전 점검만으로 통제하기 어려웠다.
6. 작업자 또는 현장 주체가 공정을 중단했다.

단, 조건 일부가 불명확하면 진성 확신도를 낮춘다.

## Step 2. 가성 강제 신호

아래 신호가 명확하면 기본적으로 가성으로 판정한다.

1. 작업 전 점검, 사전 확인, 작업 전 조치.
2. 일상적인 안전수칙 위반 또는 보호구/정리정돈/표지 등 통상 점검 이슈.
3. 행정, 승인, 서류, 절차, 교육, 허가 관련 이슈.
4. 계획된 정비, 예정된 보수, 정기 점검.
5. 협력회사 자체 시정조치 또는 작업중지권 행사 이전에 이미 통제된 사항.
6. 위험은 있으나 즉시 작업 중단이 필요한 긴급성이 부족한 사항.

## Step 3. 경계 사례 처리

텍스트에 위험 단어가 있어도 작업 전 점검이나 계획 정비 맥락이면 가성 쪽으로 본다.
반대로 조치가 단순해 보여도 작업 중 갑작스러운 낙하, 협착, 감전, 화재, 폭발, 붕괴, 유해가스, 중량물 전도 등 즉시 물리적 위험이 있으면 진성 후보로 본다.
이미지에만 핵심 위험이 있을 수 있으므로 이미지 근거가 불명확하면 review_needed를 true로 둘 수 있다.

## Step 4. 모순 방지

진성 필수조건과 가성 강제 신호가 충돌하면 다음 순서로 판단한다.

1. 작업 시점이 작업 전이면 가성을 우선한다.
2. 작업 중 실제 긴급 물리 위험이 명확하면 진성을 우선한다.
3. 행정/절차/계획정비/자체시정이면 가성을 우선한다.
4. 증거가 부족하면 판정은 진성 또는 가성 중 하나로 내리되, 확신도를 낮추고 review_needed를 true로 둔다.
